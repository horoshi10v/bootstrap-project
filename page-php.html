YII

Фреймворк Yii – це сучасний PHP фреймворк для розробки веб-застосунків будь-якого рівня складності. Yii, як і будь-який сучасний фреймворк, реалізує патерн MVC, а це означає, що логіка буде відокремлена від відображення, що зробить код більш чистим та логічним.

Yii пропонує величезну кількість додаткових можливостей, серед яких: безпека (валідація та фільтрація даних, захист від SQL-ін'єкцій та XSS), зручна робота з базами даних, Yii debug та багато іншого, що прискорює роботу зі створення сайту.

Фреймворк є full-stack фреймворком, надаючи безліч перевірених і готових до використання функцій: будівник запитів та ActiveRecord для реляційних та NoSQL баз даних, RESTful API, підтримку багаторівневого кешування тощо. Можна використовувати для створення будь-якого виду веб-додатків. Завдяки своїй основі компонентів, архітектурі та складній підтримці кешування, фреймворк підходить для розробки великомасштабних проектів.

Також Yii надає Gii – візуальний інтерфейс для автоматичної генерації контролерів, моделей та відображень.

Основні переваги та можливості фреймворку:

  • Забезпечує високу продуктивність щодо інших php-фреймворків;
  • Забезпечує високу продуктивність щодо інших php-фреймворків;
  • Дозволяє кешувати як сторінки цілком, і окремі фрагменти;
  • Здійснює перехоплення та обробку помилок;
  • Має функціонал роботи з формами, забезпечує їх побудову та валідацію;
  • Реалізовано автентифікацію та авторизацію;
  • У фреймворк вбудовані генератори базового PHP-коду для CRUD-операцій;
  • Має можливість підключення сторонніх бібліотек;
  • Працює з міграціями баз даних (генерація, застосування та відкат);
  • Підтримує стиль REST.

Приклад коду, клас віджету, в якому зберігається певна кількість функціоналу:

  class Citywidget extends CWidget{
    public $cols = 10;
    public $name;
    public $dialogId;
    public $city_id;
    public $defaultName = 'Город';

    public function init(){
        parent::init();
    }

    public function run(){
        $this->render('citywidget',[
            'name'        => $this->name,
            'dialogId'    => $this->dialogId,
            'city_id'     => $this->city_id,
            'cols'        => $this->cols,
            'defaultName' => $this->defaultName,
        ]);
    }
  }

Symfony

Це PHP-фреймворк з відкритим вихідним кодом, розроблений для веб-застосунків.

Symfony зарекомендував себе фундаментом для веб-сервісів. Він містить багаторазові бібліотеки та компоненти PHP. Такі проекти, як Laravel та Drupal, використовують компоненти Symfony у своїх додатках. Symfony був розроблений, щоб скоротити час створення та обслуговування веб-додатків та уникнути повторюваних завдань. На нього сильно вплинуло середовище Spring.

Головна ідея фреймворку Symfony у тому, що цей фреймворк має модульну структуру. Symfony працює у зв'язці з менеджером проектом Composer і дозволяє збирати проект буквально за кубиками, шматочками.

Компоненти забезпечують єдину спеціалізовану функціональність. Наприклад, компонент HTTP, компонент кешування, компонент файлової системи тощо. Це не що інше, як автономні бібліотеки PHP. Компоненти можуть отримати доступ до будь-якої кількості пакетів. Загальні пакети, що використовуються у платформі, – це FrameworkBundle, FrameworkExtraBundle, DoctrineBundle. Крім цього, у нас можуть бути власні пакети та пакети доктрин. Програма Symfony може незалежно отримувати доступ до компонентів та пакетів.

Приклад коду Symfony, компонент PropertyInfo:

  class Product {
    private $name = 'Cake';
    private $price = 10;

    public function getName() {
    return $this->name;
    }
    public function getPrice() {
    return $this->price;
    }
  }
  $class = Product::class;
  $properties = $propertyInfoObj->getProperties($class);
  /*
          Example Result
           --------------
           array(2) {
                     => string(4) "name"
                     => string(5) "price"
           }
  */

Laravel

PHP фреймворк загального призначення з відкритим кодом. Платформа використовує спільні бібліотеки з Symfony. Так як платформа має відкритий вихідний код, це надає великі можливості для кастомізації, модифікації та розширення програм. Розроблений таким чином веб-додаток є більш практичним та структурованим. Laravel дає можливість розробляти платформи, де потрібна обробка даних, великий трафік та складність.

Основні переваги Laravel:

  • велика екосистема з миттєвим розгортанням своєї платформи;
  • офіційний сайт надає безліч мануалів та інформації для ознайомлення;
  • документація Laravel близька до досконалості;
  • є власний движок для шаблонів Blade;
  • «гарний» синтаксис мови, який сприяє виконанню необхідних завдань (наприклад, аутентифікації, сесії, кешування та маршрутизації RESTful);
  • власна ORM – Eloquent ORM, що пропонує прості та зручні засоби для роботи з БД;
  • чудова система маршрутизації запитів.

Laravel – перший фреймворк, що підтримує PSR-4. Прямо із коробки Composer автоматично завантажить усі класи з каталогу app, використовуючи стандарт автозавантаження PSR.

Це означає, що у вас може бути один простір імен для вашої програми, і ви можете структурувати його, як захочете і як вважаєте логічним. Laravel не вимагає від вас розміщення певних файлів у певних папках.

Приклад коду Laravel:

  <target name="db-create">
    <echo msg="Create database '${mysql.db}'"/>
      <pdosqlexec url="mysql:host=${mysql.host};port=${mysql.port};charset=utf8"
        userid="${mysql.user}"

        DROP DATABASE IF EXISTS ${mysql.db};
        CREATE DATABASE ${mysql.db} DEFAULT CHARACTER SET 'utf8' COLLATE 'utf8_general_ci';
      </pdosqlexec>
  </target>
